Task: Revisar requisitos
En esta tarea se revisaran los requisitos antes presentados
Purpose
El propósito de revisar los requisitos en un proyecto de software es asegurar su calidad, precisión y adecuación antes de avanzar en el desarrollo del software
Relationships
RolesPrimary Performer:
  • Gerente del proyecto
Additional Performers:
InputsMandatory:
  • Requisitos
    		Optional:
    • Guia de requisitos finalizado
      Outputs
      • Funcionalidad desarrollada
        Main Description
        La revisión de requisitos en un proyecto de software es un proceso crucial que tiene como objetivo evaluar y analizar detalladamente los requisitos establecidos para el sistema a desarrollar. Consiste en examinar y validar los requisitos para asegurar su calidad, precisión y adecuación antes de proceder con el diseño y la implementación del software.
        Durante la revisión de requisitos, se lleva a cabo un análisis exhaustivo de cada requisito para identificar posibles errores, inconsistencias, ambigüedades o falta de claridad. Se verifican aspectos como la coherencia entre los requisitos, la alineación con los objetivos del proyecto y las necesidades del negocio, y la factibilidad técnica.
        Steps
        Obtener los requisitos
        Solicita y recopila todos los documentos y especificaciones que contengan los requisitos del proyecto. Estos pueden incluir documentos de alcance, casos de uso, historias de usuario, diagramas de flujo, entre otros.
        Analizar los requisitos
        Examina detenidamente cada requisito para comprender su naturaleza, su importancia y su relación con el módulo de seguridad y el login. Identifica los requisitos funcionales (qué debe hacer el sistema) y los requisitos no funcionales (atributos de calidad, restricciones técnicas, etc.).
        Validar los requisitos
        Verifica la coherencia, completitud y consistencia de los requisitos. Asegúrate de que sean comprensibles y medibles, evitando ambigüedades o contradicciones. Si es necesario, realiza consultas con los stakeholders para aclarar cualquier duda.
        Priorizar los requisitos
        Clasifica los requisitos según su importancia y urgencia. Determina cuáles son los más críticos para el módulo de seguridad y el login, y cuáles pueden abordarse en etapas posteriores.
        Identificar riesgos y desafios
        Evalúa los posibles riesgos y desafíos asociados con los requisitos de seguridad y login. Identifica las áreas que podrían presentar dificultades o implicar un mayor esfuerzo en términos de implementación, rendimiento o seguridad.
        Realizar recomendaciones
        Basándote en tu análisis, realiza recomendaciones para mejorar los requisitos en relación con la seguridad y el login. Puedes sugerir cambios, adiciones o aclaraciones que fortalezcan la robustez y la eficacia del módulo de seguridad.
        Seguimiento y revision
        Asegúrate de que se realicen los cambios y mejoras acordados en los requisitos de seguridad y login. Mantén un seguimiento continuo para verificar que los requisitos se implementen adecuadamente y se cumplan los objetivos de seguridad establecidos.
        Key Considerations
        Claridad y comprension, Consistencia y coherencia, Alineacion con las necesidades del negocio, Factibilidad tecnica, Verificabilidad
        Alternatives
        Listas de verificacion, Prototipado rápido, Revision automatizada
