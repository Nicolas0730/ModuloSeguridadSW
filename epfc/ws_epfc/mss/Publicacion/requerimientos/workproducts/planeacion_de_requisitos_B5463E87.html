Artifact: Planeacion de requisitos
La planeación de requisitos en un proyecto de software es el artefacto que establece los pasos y enfoques para identificar, analizar y documentar los requisitos del sistema
Purpose
El propósito de la planeación de requisitos es establecer una base sólida para comprender y definir los objetivos, funcionalidades y restricciones del sistema.

Description
Main DescriptionLa planeación de requisitos es un artefacto fundamental en un proyecto de software que abarca la estrategia y las actividades necesarias para identificar, analizar y documentar los requisitos del sistema. Esta etapa implica la definición de un plan que describe cómo se llevará a cabo el proceso de captura de requisitos, qué técnicas y herramientas se utilizarán, y cómo se gestionarán los cambios y actualizaciones en los requisitos a lo largo del proyecto.
Key Considerations

Definicion del alcance, Identificacion de partes interesadas, Gestion del cambio de requisitos.
